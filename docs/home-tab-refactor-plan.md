## Task 2 — Replace the Card Carousel with a 2×2 Grid

### Current behaviour (lines ~1681–2196 of `App.js`)
The "Featured Projects" section lives inside a `screen3Ref` div with `height: 400vh`. A sticky inner container holds:
- A section header ("Featured Projects")
- A `projectCarouselRef` div containing a **3-D perspective carousel** driven by `activeProjectIndex` state
- A fixed side-slider / dot indicator (right side of screen)
- Scroll-lock logic (`projectWheelLockRef`) that intercepts wheel events to step through cards

The four project cards each use a `transform: translate3d(...)` with perspective depth to simulate a rotating 3-D stack. Cards have a glow `boxShadow` and a border that brightens on hover.

### Target behaviour
Replace the 3-D carousel with a **2×2 CSS grid** of static cards. All existing card content and the cyan/purple glow effect must be retained.

### Detailed steps

#### 2.1 — Remove scroll-lock / wheel-intercept logic
In the `useEffect` that attaches a `wheel` event listener referencing `projectWheelLockRef` (around line 765), **remove or gate** the block so it no longer intercepts scroll events when the grid is rendered. The grid does not need sequential navigation.

The specific block to remove/disable:
```js
// Around line 765–830 in App.js
if (!projectCarouselRef.current || !screen3Ref.current || activeLens) return;
// ... wheel handler that calls setActiveProjectIndex
```
Delete the entire `useEffect` block or add an early return guard.

#### 2.2 — Remove the side-slider and dot indicators
The fixed side slider / project counter (lines ~1788–1949) is tied to the carousel's `activeProjectIndex`. It should be removed in full:
- The outer `position: fixed` div containing the slider track, thumb, dot indicators, and `01 / 04` counter.

#### 2.3 — Reduce the section height
The section wrapper currently uses `height: 400vh` to provide scroll room for the carousel lock. Change it to `height: auto` (or `min-height: 100vh` if you want background continuity):

```js
// Line ~1687 — screen3Ref wrapper
height: "auto",          // was "400vh"
minHeight: "100vh",      // optional — keeps dark background full-height
```

#### 2.4 — Remove the sticky wrapper
The inner `position: sticky` container is needed only for the scroll-lock effect. Replace it with a plain block-level wrapper (`position: relative`, no `sticky`, no `height: 100vh`):

```js
// Lines ~1703–1713 — inner sticky div
position: "relative",    // was "sticky"
// remove: top: 0, height: "100vh"
```

#### 2.5 — Replace the 3-D carousel markup with a 2×2 grid
Remove the `perspective` wrapper div and the `projects.map(...)` that renders offset cards.

Replace with a grid container:
```jsx
<div
  style={{
    display: "grid",
    gridTemplateColumns: "1fr 1fr",
    gap: "2rem",
    padding: "2rem 6rem 4rem",
    maxWidth: "1400px",
    margin: "0 auto",
  }}
>
  {projects.map((project, idx) => (
    <div
      key={idx}
      className="project-card"
      onMouseEnter={() => setHoveredProject(idx)}
      onMouseLeave={() => setHoveredProject(null)}
      style={{
        position: "relative",
        borderRadius: "16px",
        overflow: "hidden",
        aspectRatio: "380 / 260",       // same proportions as original card
        border: `2px solid ${
          hoveredProject === idx
            ? "rgba(137, 207, 240, 0.8)"
            : "rgba(137, 207, 240, 0.5)"
        }`,
        background: "rgba(15, 5, 30, 0.8)",
        backdropFilter: "blur(1px)",
        boxShadow:
          hoveredProject === idx
            ? "0 30px 80px rgba(137, 207, 240, 0.6), 0 0 100px rgba(139, 92, 246, 0.5)"
            : "0 20px 60px rgba(137, 207, 240, 0.4), 0 0 80px rgba(139, 92, 246, 0.3)",
        transition: "all 0.6s cubic-bezier(0.25, 0.46, 0.45, 0.94)",
        cursor: "pointer",
      }}
      onClick={() => { if (project.route) navigate(project.route); }}
    >
      {/* Video background — keep ProjectVideo component as-is, always active */}
      <ProjectVideo src={project.video} isActive={true} scale={project.scale} blur={project.blur} />

      {/* Overlay gradient — retained from original */}
      <div style={{
        position: "absolute", inset: 0,
        background: hoveredProject === idx
          ? "linear-gradient(135deg, rgba(15,5,30,0.5), rgba(25,10,50,0.6))"
          : "linear-gradient(135deg, rgba(15,5,30,0.6), rgba(25,10,50,0.7))",
        transition: "background 0.6s ease",
      }} />

      {/* Card content — identical inner layout to original */}
      <div style={{
        position: "relative", zIndex: 10, height: "100%",
        display: "flex", flexDirection: "column", justifyContent: "space-between",
        padding: "2rem",
        transform: hoveredProject === idx ? "translateY(-5px)" : "translateY(0)",
        transition: "transform 0.6s ease",
      }}>
        {/* ... project label, name, subtitle, description, View Project button — unchanged */}
      </div>
    </div>
  ))}
</div>
```

#### 2.6 — State and ref cleanup

- `projectWheelLockRef`
- `sliderPosition`

Remove only if you want a clean codebase; they are not blockers.

**Fix:** Already covered in Task 2.3 above — set `height: auto`. This collapses the empty scroll buffer entirely.

Additionally, the `projectCarouselRef` wrapper currently has:
```js

```
Confirm this is `0` (or remove it) after the grid refactor.

**Current footer padding** (line ~2212):
```js

```

**Fix:** Reduce top padding to `3rem` (or `2rem`) to close the gap:


Additionally, there is a `borderTop` on the contact div (line 2213). No change needed there — keep it as visual separation.

The `screen2Ref` (Our clients, "offerings") section currently has:

This is already `0`, so no change is needed here, but verify after implementation.

## Risk & Considerations

> [!WARNING]
> The `projectWheelLockRef` wheel listener also gates `isHoveringNav`. Confirm this flag is not relied upon elsewhere before removing the entire `useEffect`.

> [!NOTE]
> `ProjectVideo` with `isActive={true}` will autoplay all four videos simultaneously. If performance is a concern, use an `IntersectionObserver` to only play the video whose card is in the viewport. For now, all videos are small and short-looping, so simultaneous play should be acceptable.

> [!TIP]
> After the refactor, `activeProjectIndex` state can be repurposed to track a keyboard-focusable card index for accessibility, or removed entirely if not needed.
